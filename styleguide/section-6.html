6.0 - Buttons

Button styling that can be applied to any element.

.button--primary - Use when a form has a single submission button or more, emphasize the primary button. Recommendation. Only apply this style once on the screen. Always apply this to the primary "save" or "continue" button. When the primary button only appears in some form states, do not emphasize another button instead. In these cases, allow the form in such states to have no emphasized button .button--small - Sometimes tight spaces recommend the use of smaller controls. Such situations include WYSIWYG toolbars, filter forms and table rows. .button--danger - Use when you provide a button to delete the item you are editing. Recommendation. Only supply one delete button on the screen. Always supply this styling for delete buttons. .is-disabled - Use when you wish to prevent submission, but make it available later. Recommendation. Only use this when absolutely necessary, disabled buttons create cognitive dissonance. Don't apply this to the primary "save" or "delete" buttons.

Default styling

 
<button class="button {$modifiers}">Action</button>
